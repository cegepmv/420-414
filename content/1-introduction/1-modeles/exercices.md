## 1 – Identifier les modèles
Pour chacun des éléments suivants, indiquez :
+ s’il s’agit d’un modèle IaaS, PaaS ou SaaS,
+ et justifiez brièvement votre réponse.

1. Amazon EC2
2. Google App Engine
3. Microsoft Azure
4. Zoom
5. Amazon Elastic Beanstalk
6. Dropbox
7. Une machine virtuelle sur laquelle vous installez manuellement un serveur web
8. Un outil de messagerie accessible via un navigateur web

## 2 – Question de réflexion
Expliquez pourquoi un développeur pourrait préférer un modèle PaaS plutôt qu’un modèle IaaS pour un projet à court terme.


{{% expand title="Solutions" %}}
#### Exercice 1

1. IaaS

**Justification :** L’utilisateur gère le système d’exploitation, les applications et les données. AWS fournit l’infrastructure sous-jacente.

2. PaaS

**Justification :** La plateforme fournit l’environnement d’exécution. Le développeur se concentre uniquement sur le code.

3. IaaS (principalement)

**Justification :** Azure est un fournisseur cloud proposant de l’IaaS, du PaaS et du SaaS. S'il fallait choisir qu'une réponse, ce serait IaaS.

4. SaaS

**Justification :** Application prête à l’emploi accessible via Internet, sans gestion technique pour l’utilisateur.

5. PaaS

**Justification :** AWS gère l’infrastructure et la plateforme. L’utilisateur déploie son application.

6. SaaS

**Justification :** Service de stockage clé en main, accessible via navigateur ou application.

7. IaaS

**Justification :** L’utilisateur est responsable de l’OS, des logiciels et de la configuration.

8. SaaS

**Justification :** Aucune gestion d’infrastructure ou de logiciel côté utilisateur.

#### Exercice 2
+ Déploiement rapide
+ Moins de configuration et de maintenance
+ Réduction du temps consacré à l’infrastructure
+ Concentration sur le développement et les fonctionnalités
